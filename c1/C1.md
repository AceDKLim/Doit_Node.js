# 1장 - 노드에 대해 알아보고 개발 도구 설치하기

## 1. 노드란 무엇인가?

노드 - 자바스크립트 이용 서버 개발 도구

노드 만든 이유 - 동기적 입출력 서버의 시간 지연 문제 해결 위해 개발

### 동기 입출력

프로그램 → 서버 요청 → 대기 → 파일 처리 → 데이터 처리 → 다른 작업 실행

### 비동기 입출력

프로그램 → 서버 요청 → 서버 요청 기다리며 다른 작업 수행

### 노드에서 구현 이벤트 기반 입출력 방식

파일 시스템, 이벤트와 함께 호출 → 이벤트 기반 입출력

자바스크립트 → 코드 한 줄씩 해석, 인터프리터 방식 사용 → 속도 느림

but 크롬 v8 자바스크립트 엔진 → 자바스크립트를 네이티브로 변경 후 실행

노드로 프로그램 만든 후 실행 → 크롬 V8엔진 위에서 실행, 필요한 기능 병렬 실행(스레드 풀), 이벤트 받아 처리(이벤트 루프) 기본기능 + 네트워킹 기능 담당 소켓, http 라이브러리 + 표준 라이브러리 위에서 노드 프로그램 동작

노드 프로그램 동작 아키텍쳐에서 이벤트 받아 처리 기능 가장 중요. 소켓이나 HTTP 프로토콜 사용해 데이터 송수신 기능 포함 → 데이터 송수신 부분에도 이벤트 처리 방식 그대로 사용 가능

http 요청, 응당 코드

```jsx
http.reqest(option,function(res){
	res.on('data',function(chunk){
		console.log('BODY:'+chunk);
	});
});
```

http객체 → HTTP 프로토콜로 request()함수 이용해 웹 서버에 데이터 요청 → 웹 서버의 응답 받으면 콜백 함수 자동 호출 후 res객체 전달, 수신한 데이터 이벤트 이름으로 구별(여기선 data)

js에선 on()메소드 이용해 콜백 함수와 바인딩. 

여기선 res객체에 on()메소드 이용해 data이벤트와 콜백 함수 바인딩 → data이름의 이벤트 받았을 때 등록한 콜백 함수 실행, 이벤트 이름은 data, 데이터는 chunk 변수에 들어감.

바인딩 : 버튼 클릭하면 click이벤트 발생할 때 함수 객체와 바인딩 하면 click이벤트 발생할 때 등록한 함수 객체 실행.  [객체].on([이벤트 이름],[함수 객체]); 형태로 사용.

### 노드를 더 쉽게 사용하는 모듈

소스 코드에 실행 기능 모두 들어있으면 코드 길고 복잡해짐 → 별도의 자바스크립트 파일로 코드 분리해서 관리, 필요할 때 불러서 사용.

node는 CommonJs 표준 스펙 따라 만듦.

별도의 파일 → 모듈,  require() 함수로 호출. 객체로 참조해 사용. 

여러 개의 모듈 → 패키지 → 다른 사람이 npm으로 손쉽게 설치 가능
